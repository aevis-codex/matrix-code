# MatrixCode 第四阶段受控本地执行代理设计规格

## 概述

第四阶段把第三阶段的模型网关和角色工作台连接到一个受控本地执行代理。系统需要让项目登记本地工作区，在授权根路径内读取文件、写入小型文本文件、执行白名单测试命令、采集 Git diff 摘要，并把执行结果、审批结果和审计摘要展示到工作台与桌面端右侧指标栏。

本阶段坚持“真实能力、有限边界”。文件和命令操作会落到本机工作区，但所有入口都必须经过工作区授权、路径约束、审批策略和审计脱敏。SSH、远程部署和生产环境操作只生成审批记录，不建立真实远程连接。

## 目标

- 为项目登记一个或多个授权本地工作区，并记录最近一次代理心跳。
- 在授权工作区内列目录、读文件和写入小型文本文件。
- 对路径做规范化校验，阻止绝对路径逃逸、`..` 路径穿越和符号链接逃逸。
- 执行明确安全的本地测试命令，并记录退出码、标准输出摘要、标准错误摘要和耗时。
- 对危险命令、SSH、部署、凭证和未知命令返回 `ASK` 或 `DENY`，不直接执行。
- 采集授权工作区的 Git diff 文件列表和统计摘要。
- 将执行代理摘要接入 `GET /api/projects/{projectId}/workbench`。
- 桌面端展示授权工作区、最近执行、审批状态、Git diff 摘要和审计摘要。
- 所有接口、文档和界面文案保持中文。

## 非目标

- 不实现真实 SSH 连接、SCP、rsync、远程部署、回滚或健康检查探测。
- 不执行任意 Shell 脚本、管道、重定向、后台进程或交互式命令。
- 不读取密钥文件，不展示完整环境变量，不接受明文凭证。
- 不实现数据库持久化、登录鉴权、多人权限模型或组织级审计报表。
- 不让模型网关直接调用执行代理；本阶段由显式 API 和桌面按钮触发。
- 不实现长任务队列、并发调度、取消任务或流式日志。

## 用户流程

开发进入工作台后，在开发面板登记本地工作区路径。服务端校验路径存在、是目录，并为项目创建授权记录。右侧指标栏显示工作区名称、根路径和最近心跳状态。

开发可以请求列出工作区内的相对路径，或者读取一个小型文本文件。服务端只接受相对路径，返回目录项或文件内容摘要。读取超过大小限制、二进制文件、未授权路径或路径逃逸时，接口返回错误。

开发可以写入一个小型文本文件，例如生成接口文档草稿或部署说明草稿。写入目标必须仍在授权根路径内，父目录必须存在；写入后发布事件，并记录审计摘要。

开发可以触发安全测试命令，例如服务端 Maven 测试或桌面端 `npm test`。审批策略返回 `ALLOW` 时，服务端在授权工作区内执行命令并保存结果。审批策略返回 `ASK` 时，只创建待审批执行记录和审计记录，不执行命令。审批策略返回 `DENY` 时拒绝请求。

运维配置 SSH 地址后，如果触发“准备部署审批”，系统只登记一条 `SSH` 或部署命令的审批记录，并在工作台显示待审批状态。真实远程连接留到后续阶段。

用户可以刷新 Git diff 摘要。服务端在授权工作区执行固定的 `git diff --name-only` 和 `git diff --stat`，返回变更文件和统计文本。该能力不接受用户自定义 Git 参数。

## 数据模型

### 授权工作区

- 工作区 ID。
- 项目 ID。
- 显示名称。
- 根路径。
- 授权状态：已授权、已撤销。
- 创建时间。
- 最近访问时间。

### 文件操作

- 操作 ID。
- 项目 ID。
- 工作区 ID。
- 操作类型：列目录、读文件、写文件。
- 相对路径。
- 结果状态。
- 内容摘要或错误摘要。
- 创建时间。

### 执行任务

- 任务 ID。
- 项目 ID。
- 工作区 ID。
- 执行人。
- 工具类型：`SHELL`、`GIT`、`SSH`。
- 命令摘要。
- 审批结果：`ALLOW`、`ASK`、`DENY`。
- 执行状态：待审批、已拒绝、执行中、成功、失败。
- 退出码。
- 标准输出摘要。
- 标准错误摘要。
- 耗时毫秒。
- 创建时间。

### Git diff 摘要

- 项目 ID。
- 工作区 ID。
- 是否为 Git 仓库。
- 变更文件列表。
- 统计摘要。
- 采集时间。

### 执行代理摘要

- 授权工作区列表。
- 最近代理心跳。
- 最近文件操作。
- 最近执行任务。
- 最近 Git diff 摘要。
- 最近审计记录。

## 服务端设计

新增 `localexecution` 模块：

- `localexecution.domain`：授权工作区、文件操作请求和结果、执行命令请求和结果、Git diff 摘要、工作台摘要。
- `localexecution.application`：工作区注册服务、路径守卫、文件服务、命令执行服务、Git diff 服务、执行摘要服务。
- `localexecution.api`：项目级本地执行代理接口。

复用现有模块：

- `approval`：继续使用 `ApprovalPolicy` 判断命令是否允许自动执行，并使用 `AuditService` 记录脱敏摘要。
- `execution`：保留代理心跳和结果上报接口；第四阶段新增的本地执行结果可以继续进入该网关，避免重复概念。
- `workbench`：聚合本地执行代理摘要，工作台返回 `localExecution` 字段。
- `realtime`：文件写入、命令完成、审批待处理和 Git diff 采集后发布中文事件。

### 路径守卫

路径守卫只接受相对路径。校验流程：

1. 工作区根路径必须存在且是目录。
2. 用户输入路径不能是绝对路径，不能为空，不能包含空路径段。
3. 使用 `root.resolve(relativePath).normalize()` 得到目标路径。
4. 目标路径必须以工作区根路径开头。
5. 读取已有文件时使用真实路径校验，阻止符号链接逃逸。
6. 写入新文件时校验父目录真实路径仍在工作区内。

### 命令执行

命令执行服务不打开通用 Shell。它把命令按空白分词后交给 `ProcessBuilder`，工作目录固定为授权工作区根路径。第一版只自动执行审批策略返回 `ALLOW` 的命令，例如：

- `/Users/Masons/Ai/Maven/bin/mvn -Dmaven.repo.local=/Users/Masons/Ai/Maven_Ai_Store -q -pl server test`
- `npm test`
- `pnpm test`
- `yarn test`

命令超时时间默认 60 秒。标准输出和标准错误分别截断到 4 KB。退出码为 0 记为成功，非 0 记为失败。`ASK` 和 `DENY` 不启动进程。

### Git diff

Git diff 服务只执行固定命令：

- `git diff --name-only`
- `git diff --stat`

如果工作区不是 Git 仓库，返回 `isRepository=false` 和空变更列表。服务不接受用户提供的 Git 参数，不执行 `git reset`、`git clean`、`git checkout` 或提交操作。

## API 设计

- `POST /api/projects/{projectId}/local-execution/workspaces`：登记授权工作区。
- `GET /api/projects/{projectId}/local-execution/summary`：查询本地执行代理摘要。
- `POST /api/projects/{projectId}/local-execution/files/list`：列出目录。
- `POST /api/projects/{projectId}/local-execution/files/read`：读取文本文件。
- `POST /api/projects/{projectId}/local-execution/files/write`：写入文本文件。
- `POST /api/projects/{projectId}/local-execution/commands`：提交命令执行请求。
- `POST /api/projects/{projectId}/local-execution/git-diff`：采集 Git diff 摘要。
- `GET /api/projects/{projectId}/workbench`：继续返回工作台聚合视图，并新增本地执行代理摘要。

所有写接口都要求 `projectId` 与工作区所属项目一致。`workspaceId` 不存在、已撤销或不属于项目时返回错误。

## 桌面端设计

开发面板继续保留现有“本地工作区路径”输入。第四阶段新增轻量操作区：

- 授权当前工作区。
- 读取目录或文件。
- 写入执行说明草稿。
- 运行安全测试命令。
- 刷新 Git diff。

右侧指标栏新增本地执行代理区域：

- 授权工作区。
- 最近命令状态和审批结果。
- 最近 Git diff 文件数。
- 最近审计摘要。
- SSH 或部署审批待处理提示。

界面不提供任意命令历史输入框。命令入口使用预设按钮或明确的安全命令文本，避免把桌面端变成通用终端。

## 状态与事件

新增事件类型：

- `LOCAL_WORKSPACE_AUTHORIZED`
- `LOCAL_FILE_READ`
- `LOCAL_FILE_WRITTEN`
- `LOCAL_COMMAND_APPROVAL_PENDING`
- `LOCAL_COMMAND_COMPLETED`
- `LOCAL_COMMAND_REJECTED`
- `LOCAL_GIT_DIFF_CAPTURED`

事件消息必须使用中文，包含角色或执行人、工作区名称、审批结果和状态摘要。审计记录中的敏感参数必须脱敏后再进入工作台展示。

## 权限与安全

- 所有本地路径能力必须先登记授权工作区。
- API 不接受绝对文件路径作为文件操作目标。
- 不读取超过 64 KB 的文件，不写入超过 128 KB 的内容。
- 不读取明显的二进制文件。
- 不自动执行 `ASK` 或 `DENY` 命令。
- 不执行包含管道、重定向、命令连接符、子 Shell、后台进程或环境变量赋值的命令。
- SSH、部署、凭证、删除、清理、回滚和基础设施命令只能进入审批记录。
- 审计摘要必须复用现有脱敏策略。

## 测试策略

### 服务端

- 工作区注册测试：存在目录可以授权；不存在路径、文件路径和空项目编号被拒绝。
- 路径守卫测试：相对路径通过；绝对路径、`..`、符号链接逃逸和未授权工作区被拒绝。
- 文件服务测试：列目录、读文本、写小文件通过；大文件和二进制文件被拒绝。
- 命令服务测试：安全测试命令执行并记录结果；危险命令返回待审批；空命令或逃逸工作区被拒绝。
- Git diff 测试：Git 仓库返回文件列表和统计摘要；非 Git 目录返回非仓库状态。
- 工作台测试：工作台返回 `localExecution` 摘要、最近命令和审计记录。
- 接口测试：授权工作区、文件读取、命令请求、Git diff 和工作台聚合接口可串起来。

### 桌面端

- API 客户端测试：覆盖工作区授权、摘要读取、文件读取、命令提交和 Git diff。
- 工作台展示测试：右侧指标栏展示授权工作区、最近命令、审批状态、Git diff 和审计摘要。
- 开发流程回归测试：提交开发交付仍能生成文档和事件；新增本地执行摘要不破坏现有表单。

## 验收标准

- 服务端全量测试通过，命令使用 `/Users/Masons/Ai/Maven/bin/mvn -Dmaven.repo.local=/Users/Masons/Ai/Maven_Ai_Store -q -pl server test`。
- 桌面端测试、构建和 Tauri 命令入口通过。
- `POST /api/projects/demo/local-execution/workspaces` 可以登记当前项目工作区。
- 文件读取和写入只能在授权工作区内完成，路径逃逸请求被拒绝。
- 安全测试命令可以执行并返回退出码与输出摘要。
- 危险命令、SSH 和部署命令只返回待审批或拒绝，不执行真实动作。
- `POST /api/projects/demo/local-execution/git-diff` 返回当前工作区变更摘要。
- `GET /api/projects/demo/workbench` 返回 `localExecution` 字段，并且桌面端右侧栏能展示。
- 文档中没有占位符、英文模板标题或不符合本地 Maven 路径的命令。

## 后续阶段入口

后续阶段可以在本阶段边界上继续扩展：

- 人工审批 UI 和审批通过后的二次执行。
- 真实 SSH 执行、健康检查探测和回滚。

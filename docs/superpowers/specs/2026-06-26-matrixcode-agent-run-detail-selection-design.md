# MatrixCode Agent 运行详情选择设计

## 背景

第 61 阶段已经让模型请求完成事件进入 Agent Runtime 时间线。运行中心仍默认围绕最新运行展示详情，历史运行虽然已被加载，但用户无法直接切换查看某次历史运行的模型请求、工具 trace 和失败事件。

## 目标

在运行中心提供轻量 Agent 运行选择器。用户选择某次运行后，Agent 审计详情、事件计数、事件筛选、模型请求关联和模型请求 trace 展示全部跟随选中运行变化。

## 推荐方案

- 运行中心新增 `Agent 运行选择` 卡片，横向/网格展示最近加载的运行。
- 默认选中最新运行；选择的运行 ID 保存在 `OperationsDetailPanel` 本地状态中。
- Agent 审计详情使用选中运行，而不是固定使用 `agentRuns[0]`。
- Agent 事件时间线先按选中运行过滤，再按“全部 / 工具 / 失败”筛选。
- 模型请求关联按选中运行匹配 `recentRequests.agentRunId`。
- `TOOL_TRACE` 渲染增加模型请求 metadata 展示：供应商、模型、缓存来源、稳定前缀指纹、缓存策略和 volatile 后缀策略。
- 真实运行联调中发现桌面端默认后端地址仍指向 `http://localhost:18080`，而 Spring Boot 默认端口和多数真实验证脚本使用 `8080`；本阶段顺手将默认桌面端 API 地址和 `.env.example` 统一为 `http://localhost:8080`。

## 边界

- 不新增后端 API；继续复用已经加载的最近运行和事件。
- 不改右侧常驻运行指标；右侧仍展示最新运行摘要，避免主工作区变复杂。
- 不新增数据库结构。
- 不展示 prompt、模型响应正文、工具输出、完整文件内容、API Key 或数据库密码。
- 端口统一只调整默认值，仍允许通过 `VITE_MATRIXCODE_SERVER_URL` 和 `SERVER_PORT` 覆盖，避免限制部署拓扑。

## 验收标准

- 运行中心出现 `Agent 运行选择`。
- 默认展示最新运行的审计详情和该运行事件计数。
- 点击历史运行后，审计详情切换为该运行。
- 历史运行只显示自己的事件，不混入其他运行事件。
- 关联模型请求跟随选中运行变化。
- 模型请求 `TOOL_TRACE` 能展示模型、缓存来源、prefix fingerprint 和缓存策略。
- 无额外环境变量时，`npm --prefix desktop run dev` 默认连接本地 Spring Boot `8080`，工作台不再因为端口错配进入连接错误页。
- 桌面端目标测试、桌面端全量、桌面端构建、服务端全量、真实集成、静态检查和密钥扫描通过。

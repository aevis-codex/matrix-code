# MatrixCode 第五阶段人工审批执行闭环设计规格

## 概述

第五阶段把第四阶段留下的 `APPROVAL_PENDING` 本地执行任务升级为可操作的人工审批闭环。用户可以在桌面端右侧“本地执行代理”卡片中查看待审批命令，明确选择批准执行或拒绝。服务端只允许对原始待审批任务做一次审批决策，批准后执行原始命令，拒绝后写入审计并结束任务。

本阶段继续坚持“显式审批、有限执行”。人工批准不是万能越权开关：SSH、远程部署、凭证命令、危险 Shell 语法和破坏性命令仍不在第五阶段执行范围内。它们可以被记录、展示和拒绝，但真实远程执行留到后续阶段单独设计。

## 目标

- 查询项目内待审批本地执行任务。
- 对单个待审批任务做批准或拒绝决策。
- 批准后只执行任务原始命令，不接受新命令参数。
- 拒绝后任务进入 `DENIED`，并记录拒绝人和审计摘要。
- 防止同一任务被重复批准、重复拒绝或重复执行。
- 对人工批准仍做二次安全校验，禁止 SSH、部署、凭证和高风险命令落到真实进程。
- 在工作台 `localExecution` 摘要中展示待审批队列、最近决策和执行结果。
- 桌面端右侧指标栏内联展示审批队列，提供“批准执行”和“拒绝”按钮。
- 所有文案、错误和审计摘要保持中文。

## 非目标

- 不实现真实 SSH、SCP、rsync、远程部署、回滚或生产环境操作。
- 不实现多人审批、审批会签、权限角色和登录鉴权。
- 不实现批量审批。
- 不实现任务取消、流式日志或长任务后台队列。
- 不把桌面端改造成通用终端，不新增任意命令输入框。
- 不实现数据库持久化；第五阶段继续使用内存仓储完成纵切验证。

## 用户流程

开发或运维触发一个需要审批的本地执行命令后，服务端返回 `APPROVAL_PENDING`。右侧“本地执行代理”卡片显示该任务的执行人、命令摘要、审批结果、风险提示和创建时间。

用户点击“拒绝”时，桌面端调用审批决策接口。服务端校验任务仍处于 `APPROVAL_PENDING`，将任务状态更新为 `DENIED`，把审批结果更新为 `DENY`，记录审计事件，并刷新工作台。任务不会启动进程。

用户点击“批准执行”时，桌面端调用同一个审批决策接口，决策为 `ALLOW`。服务端校验任务仍处于 `APPROVAL_PENDING`，再次校验工作区仍授权，并对原始命令执行人工批准安全检查。通过检查后，服务端在原授权工作区内执行原始命令，记录退出码、输出摘要、错误摘要和耗时。未通过安全检查时，任务进入 `DENIED`，错误摘要说明拒绝原因，并记录审计。

任务完成后，右侧卡片显示最新状态。成功和失败都算审批闭环完成；失败不自动重试。用户如需再次执行，必须重新提交一个新的命令请求。

## 命令批准边界

第五阶段将待审批命令分为三类：

- 自动允许命令：第四阶段已有安全测试命令，例如 `npm test`、`pnpm test`、`yarn test` 和指定 Maven 测试命令。它们提交时直接执行，不进入人工审批。
- 可人工批准的本地命令：不包含 Shell 连接符、管道、重定向、后台执行、子 Shell、环境变量赋值、凭证参数、绝对路径逃逸或破坏性动词的本地命令，例如 `npm run test:integration`、`git status`、`git diff --stat`、`./gradlew build`。
- 不可人工批准执行的命令：SSH、部署、回滚、删除、清理、权限修改、凭证读取、密钥文件、生产环境地址和基础设施命令。这类命令可以进入待审批记录，但第五阶段批准时仍会被拒绝执行。

人工审批接口不接收命令正文。它只能引用已有 `taskId`，因此无法绕过提交阶段的审计脱敏和工作区绑定。

## 数据模型

### 执行任务扩展

沿用第四阶段 `ExecutionTask`，新增审批决策字段：

- 审批人 ID。
- 审批备注。
- 审批决策时间。
- 安全拒绝原因。

`approvalDecision` 表示任务当前决策状态。任务初次提交为 `ASK` 时，状态是 `APPROVAL_PENDING`。人工拒绝后，`approvalDecision` 更新为 `DENY`，状态更新为 `DENIED`。人工批准并执行后，`approvalDecision` 更新为 `ALLOW`，状态更新为 `SUCCESS` 或 `FAILED`。

### 审计记录

审批动作继续复用 `AuditService`，并为同一任务记录两类审计：

- 提交审计：命令第一次进入审批策略时记录 `ASK`。
- 决策审计：用户批准或拒绝时记录 `ALLOW` 或 `DENY`。

审计摘要必须继续脱敏凭证、Token、密钥、用户名参数和 SSH 私钥路径。

### 本地执行摘要

`LocalExecutionSummary` 继续返回最近任务和最近审计记录。待审批队列可以从 `recentTasks` 中筛选 `status=APPROVAL_PENDING` 得到。第五阶段不额外引入单独队列对象，避免重复概念。

## 服务端设计

扩展 `LocalCommandService`：

- `decide(projectId, taskId, actorId, decision, note)`：处理人工审批。
- 只允许 `ALLOW` 或 `DENY` 作为人工决策。
- 按项目查找原始任务，任务不存在时报中文错误。
- 任务不属于当前项目时报中文错误。
- 任务状态不是 `APPROVAL_PENDING` 时报中文错误，防止重复执行。
- 工作区已撤销或不存在时，批准失败并返回中文错误。
- 人工批准前调用安全校验，确认原始命令属于可人工批准的本地命令。
- 批准通过后复用第四阶段 `ProcessBuilder` 执行逻辑。
- 拒绝或安全拒绝都记录审计。

扩展 `LocalExecutionController`：

- 新增 `POST /api/projects/{projectId}/local-execution/commands/{taskId}/approval`。
- 请求体包含 `actorId`、`decision` 和可选 `note`。
- 响应返回更新后的 `ExecutionTask`。

审批接口的 `decision=ALLOW` 不是直接信任用户输入。服务端仍拥有最终执行权；凡是第五阶段不允许执行的命令，均返回 `DENIED` 或中文错误，并写入审计。

## 桌面端设计

右侧 `InspectorPanel` 的“本地执行代理”卡片升级为内联审批队列：

- 显示授权工作区名称和根路径。
- 显示最近待审批任务，内容包括执行人、状态、命令摘要和风险提示。
- 待审批任务提供两个按钮：“批准执行”和“拒绝”。
- 操作中按钮禁用，避免重复点击。
- 决策完成后刷新工作台，卡片显示最新任务状态、退出码和审计摘要。
- 无待审批任务时，展示最近命令状态、Git diff 文件数和最近审计。

桌面 API 客户端新增：

- `decideLocalCommandApproval(projectId, taskId, input, serverUrl)`。

`App.tsx` 负责把审批处理函数传入 `InspectorPanel`，并在成功后调用 `refreshWorkbench({ keepCurrent: true })`。同步失败沿用现有 `syncFailureMessage`。

## 状态与事件

新增或复用以下事件：

- `LOCAL_COMMAND_APPROVED`：用户批准待审批命令。
- `LOCAL_COMMAND_DENIED`：用户拒绝待审批命令。
- `LOCAL_COMMAND_COMPLETED`：批准后的命令执行完成。
- `LOCAL_COMMAND_REJECTED`：安全策略拒绝人工批准执行。

事件消息使用中文，包含审批人、命令摘要和结果。事件中不展示未脱敏凭证。

## 错误处理

- 任务不存在：`待审批任务不存在`。
- 任务已决策：`任务已完成审批，不能重复处理`。
- 决策为空或不是 `ALLOW` / `DENY`：`审批决策只能是批准或拒绝`。
- 执行人为空：`审批人不能为空`。
- 工作区未授权：沿用第四阶段工作区错误。
- 命令不允许人工批准执行：`该命令不在第五阶段可批准执行范围内`。

桌面端显示服务端返回的中文错误。操作失败后不清空现有工作台数据。

## 测试策略

### 服务端

- 命令审批测试：待审批本地命令可以被拒绝，状态变为 `DENIED`，审计记录追加 `DENY`。
- 命令执行测试：可人工批准的本地命令批准后执行原始命令，返回退出码和输出摘要。
- 重复审批测试：已拒绝或已执行任务再次审批会失败，不重复执行。
- 安全边界测试：SSH、部署、凭证和危险 Shell 语法命令即使人工批准也不会执行。
- 工作区校验测试：工作区撤销后，待审批任务不能被批准执行。
- 接口测试：审批决策接口返回更新后的任务，并通过工作台摘要展示最新状态。

### 桌面端

- API 客户端测试：审批接口 URL、方法和请求体正确。
- 指标栏测试：待审批任务展示“批准执行”和“拒绝”按钮。
- 交互测试：点击批准或拒绝会调用审批接口并刷新工作台。
- 错误测试：审批失败时显示同步错误，并保留当前工作台内容。

## 验收标准

- 服务端全量测试通过，命令使用 `/Users/Masons/Ai/Maven/bin/mvn -Dmaven.repo.local=/Users/Masons/Ai/Maven_Ai_Store -q -pl server test`。
- 桌面端测试、构建和 Tauri 命令入口通过。
- 提交未知但可人工批准的本地命令后，工作台显示 `APPROVAL_PENDING`。
- `POST /api/projects/demo/local-execution/commands/{taskId}/approval` 使用 `DENY` 后返回 `DENIED`，且不会执行命令。
- 同一接口使用 `ALLOW` 后只执行原始命令，并返回 `SUCCESS` 或 `FAILED`。
- 同一个任务第二次审批返回中文错误，不重复执行。
- SSH 或部署命令即使人工批准，也不会触发真实远程动作。
- 桌面端右侧“本地执行代理”卡片能完成单个待审批任务的批准和拒绝操作。
- 文档中没有占位符、英文模板标题或不符合本地 Maven 路径的命令。

# MatrixCode 第三阶段模型网关与缓存优化设计规格

## 概述

第三阶段把第二阶段的确定性产品草稿生成器升级为可替换的模型网关纵切。系统需要支持模型供应商配置、角色模型选择、稳定提示词前缀、请求上下文清单、prompt cache 命中估算、token 成本统计，并把这些指标展示到服务端工作台和桌面端右侧指标栏。

本阶段仍然保持可离线验证。默认模型适配器使用本地确定性实现，保证没有 API Key、没有外网、没有真实费用时也能完整跑通服务端测试、桌面端测试和运行态接口。真实 OpenAI 兼容供应商通过接口和配置形态预留，不作为默认测试依赖。

## 目标

- 管理项目级模型供应商配置，至少支持本地确定性供应商和 OpenAI 兼容供应商描述。
- 为产品、开发、测试、运维四个角色分别选择模型、价格和上下文策略。
- 为每次角色模型请求生成稳定提示词前缀、工具契约版本和上下文清单。
- 估算 prompt cache 命中 token、未命中输入 token、输出 token、缓存命中率和费用。
- 将请求摘要、模型选择、上下文块、缓存指标和费用写入项目事件与工作台指标。
- 桌面端展示当前角色模型、缓存命中率、词元、费用、上下文块和最近模型请求。

## 非目标

- 不存储真实供应商 API Key，不实现组织级密钥加密。
- 不接入真实流式响应。
- 不实现供应商重试、限流、熔断或多供应商路由。
- 不把模型输出直接写入本地工作区。
- 不实现数据库持久化、登录鉴权、多租户隔离。
- 不进入第四阶段的本地执行代理、文件读写、命令执行、SSH 或部署动作。

## 用户流程

项目创建后，系统默认注册一个本地确定性供应商 `local-deterministic`，并给四个角色绑定默认模型。用户在工作台中可以看到每个角色当前使用的模型、货币、价格口径和缓存策略。

产品角色提交需求时，工作台不再直接调用产品草稿生成器，而是通过模型网关创建一次产品模型请求。模型网关组装稳定系统前缀、角色契约、工具契约、项目规则、允许进入上下文的文档和当前用户指令，然后交给本地模型适配器生成稳定中文草稿。生成结果继续落到文档中心。

开发、测试、运维角色在提交交付说明、测试报告或部署目标时，可以生成一次模型请求摘要，用于记录该角色本轮上下文包含了哪些冻结产物、事件和文档。第三阶段先不要求模型替代这些表单动作，但需要统一记录模型网关指标。

当同一角色会话多次使用相同稳定前缀时，缓存估算器把稳定前缀 token 计为命中；新的用户指令和动态上下文计为未命中。工作台展示累计缓存命中率、会话 token 和费用。

## 数据模型

### 模型供应商

- 供应商 ID：例如 `local-deterministic`、`openai-compatible`。
- 名称：用户可读中文名称。
- 协议：`LOCAL` 或 `OPENAI_COMPATIBLE`。
- 基础地址：真实供应商的兼容接口地址；本地供应商为空。
- API Key 来源：环境变量名或 `NONE`。
- 是否启用。

### 角色模型绑定

- 项目 ID。
- 角色：产品、开发、测试、运维。
- 供应商 ID。
- 模型名称。
- 币种。
- 缓存命中输入单价、未命中输入单价、输出单价。
- 上下文预算 token。
- 稳定工具契约版本。

### 提示词契约

- 角色。
- 稳定系统前缀。
- 角色输出契约。
- 工具契约版本。
- 稳定前缀哈希。
- 估算稳定前缀 token。

稳定前缀必须由可审计字段生成。相同角色、相同工具契约、相同组织规则和相同模型绑定应得到相同哈希。用户本轮输入、临时事件和动态文档摘要不能进入稳定前缀哈希。

### 模型请求

- 请求 ID。
- 项目 ID。
- 角色。
- 角色会话 ID。
- 模型绑定。
- 提示词契约。
- 上下文清单。
- 用户指令。
- 生成结果。
- 用量记录。
- 创建时间。

### 工作台模型指标

- 当前角色模型列表。
- 最近模型请求列表。
- 累计 cache hit token。
- 累计 cache miss input token。
- 累计 output token。
- 累计费用。
- 缓存命中率。
- 最近上下文块类型列表。

## 服务端设计

新增 `modelgateway` 模块：

- `modelgateway.domain`：供应商、角色绑定、提示词契约、模型请求、模型响应、模型调用结果。
- `modelgateway.application`：供应商注册、角色绑定、提示词契约生成、模型请求编排、缓存估算、请求历史查询。
- `modelgateway.api`：项目模型配置、角色绑定和角色模型请求接口。

扩展现有模块：

- `context`：保留上下文门禁能力，增加用于模型请求的动态上下文摘要字段。
- `usage`：沿用 `UsageCalculator`，由模型网关统一创建请求级 `UsageRecord`。
- `workbench`：工作台聚合模型绑定、最近模型请求和累计用量指标。
- `agent`：产品草稿生成器可以作为本地模型适配器的一部分继续复用，输出仍保持确定性。
- `realtime`：模型请求完成后发布 `MODEL_REQUEST_COMPLETED` 事件。

## API 设计

- `GET /api/projects/{projectId}/model-gateway/config`：返回供应商、角色模型绑定和累计指标。
- `POST /api/projects/{projectId}/model-gateway/providers`：注册或更新项目模型供应商配置。
- `POST /api/projects/{projectId}/roles/{role}/model-binding`：设置角色模型绑定。
- `POST /api/projects/{projectId}/roles/{role}/model-requests`：创建角色模型请求。
- `GET /api/projects/{projectId}/workbench`：继续返回工作台聚合视图，并增加模型指标字段。

第三阶段接口只接受项目内配置，不接收明文密钥。OpenAI 兼容供应商的 API Key 只允许填写环境变量名，真实读取和调用留给后续供应商适配计划。

## 桌面端设计

右侧运行指标栏增加模型网关区域：

- 当前角色模型。
- 模型供应商。
- 缓存命中 token、未命中输入 token、输出 token。
- 估算费用和币种。
- 最近模型请求摘要。
- 最近进入上下文的块类型。

产品面板继续保留“生成产品草稿”动作。用户体验不新增复杂模型配置表单，第三阶段先通过服务端默认配置和 API 客户端能力验证模型网关。后续可在设置页中补全供应商管理。

## 状态与事件

模型请求是追加记录，不覆盖历史请求。模型网关完成请求后发布事件：

- `MODEL_PROVIDER_CONFIGURED`
- `ROLE_MODEL_BOUND`
- `MODEL_REQUEST_COMPLETED`

事件消息必须使用中文，并包含角色、模型和缓存命中率摘要。失败请求在本阶段通过异常返回，不落入长任务队列。

## 权限与安全

- 不接受明文 API Key。
- 不从桌面端读取本地密钥文件。
- 模型供应商配置中的基础地址必须是 `https://`，本地确定性供应商除外。
- 角色模型绑定只能引用已启用供应商。
- 请求上下文必须先经过上下文门禁，不能直接把产品长对话、未冻结文档或无关日志塞入下游角色。

## 测试策略

### 服务端

- 供应商配置测试：默认供应商存在，禁用供应商不能绑定，OpenAI 兼容地址必须是 HTTPS。
- 角色绑定测试：四个角色有默认模型；角色绑定保存后可查询。
- 提示词契约测试：稳定字段相同则哈希相同；用户输入变化不改变稳定前缀哈希；工具契约变化会改变哈希。
- 缓存估算测试：同一角色会话第二次请求命中稳定前缀；不同角色会话不共享命中。
- 模型网关测试：创建产品模型请求后返回中文草稿、上下文清单、用量记录和事件。
- 工作台测试：工作台返回模型指标、最近模型请求和上下文块。
- 接口测试：配置、绑定、请求和工作台聚合接口可串起来。

### 桌面端

- API 客户端测试：覆盖模型配置读取、角色绑定和模型请求提交。
- 工作台展示测试：右侧指标栏展示模型名、费用、上下文块和最近请求。
- 产品流程回归测试：生成产品草稿后仍能刷新文档交接和事件流。

## 后续阶段入口

第四阶段仍建议接入真实本地执行代理：

- 工作区授权。
- 文件读写。
- 命令执行。
- Git diff 采集。
- SSH 审批与部署动作。

真实外部模型调用、供应商密钥加密、流式响应、重试限流和多供应商路由可以作为第三阶段之后的模型网关增强计划继续拆分。

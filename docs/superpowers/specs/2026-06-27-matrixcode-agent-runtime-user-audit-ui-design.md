# MatrixCode Agent Runtime 用户责任审计 UI 设计

## 背景

第 73 阶段已提供后端只读接口 `GET /api/projects/{projectId}/agent-runs/user-audit?userId=...`。当前桌面端运行中心仍只能看到运行、事件、模型请求和恢复操作，不能直接按当前用户查看责任链。第 74 阶段把该接口接入运行中心，形成阶段验收可见的责任审计入口。

## 目标

- 在桌面端运行中心展示当前操作者的 Agent Runtime 用户责任审计。
- 让用户能看到责任运行数、活跃责任数、模型请求数和责任条目。
- 明确责任来源：当前认领人、角色成员回退或运行操作者。
- 保持右侧运行指标简洁，不把责任明细塞回右侧 Inspector。

## 非目标

- 不实现登录认证、权限拦截或成员邀请流程。
- 不新增后端 DDL。
- 不在前端展示完整 prompt、模型响应、向量正文、工具输出、命令输出或密钥。
- 不新增自动执行器，不触发模型、命令、文件写入或 Patch。

## 推荐方案

采用「运行中心内嵌责任审计卡片」：

- API client 新增 `loadAgentRunUserAudit(projectId, userId)`。
- `App` 在加载工作台时并行加载当前操作者的用户责任审计。
- `OperationsDetailPanel` 只在运行中心弹窗中展示责任审计卡片。
- 接口失败时返回空报告并展示空态，避免旧服务端或网络波动影响主工作台。

## UI 结构

运行中心新增卡片：

- 标题：`用户责任审计`
- 统计：`运行 N · 活跃 M · 模型请求 K`
- 用户：`当前用户 user-dev`
- 条目列表：
  - `运行 run-1 · 认领 Worker · 运行中`
  - `目标 修复支付失败重试`
  - `角色 开发 · coding · 工具 1 · 模型请求 1`
  - `最近事件 工具调用 trace`

窄屏下列表仍为纵向卡片，不使用宽表格，避免移动视口横向溢出。

## 数据流

1. 当前操作者来自现有 `selectedActorUserId` 或角色默认用户。
2. `refreshWorkbench` 并行加载 `roleAgentConfigs`、`projectMembers`、`agentRuntime` 和 `agentRunUserAudit`。
3. 运行中心接收 `agentRunUserAudit`。
4. 如果用户切换，重新刷新工作台时重新读取审计报告。

## 错误处理

- 审计接口失败：返回空报告并在卡片显示「暂无责任运行」。
- 主工作台接口失败：保持既有错误处理。
- 审计报告和运行列表不一致：以审计报告为准展示责任链，以运行列表继续展示运行详情。

## 测试策略

- API client 测试：确认请求路径、query 和 `Accept` 头。
- App 测试：确认运行中心展示当前用户责任统计和责任条目。
- 降级测试：审计接口失败时运行中心仍可打开，并展示空态。
- 全量验证：桌面端测试、桌面构建、服务端全量、真实运行检查、真实集成、静态检查和安全扫描。

## 回溯对齐

- 对齐最初需求：多人实时协作的智能体控制台需要角色和用户维度的责任链。
- 对齐第 39 到 40 阶段：复用当前操作者和项目成员用户归属。
- 对齐第 73 阶段：前端只消费低敏审计报告，不扩大数据暴露面。
- 对齐布局要求：右侧只保留运行指标和关键事件，责任明细放入运行中心页面级弹窗。

# MatrixCode Agent trace 筛选与模型请求线索设计

## 背景

第 57 阶段已经把 `TOOL_TRACE` 原始 JSON 展示为结构化摘要。运行中心下一步需要支持更快排障：操作者应能只看工具 trace 或失败事件，并能看到当前项目最近一次模型请求线索。

## 目标

在桌面端运行中心增加 Agent 事件筛选和最近模型请求线索，提升运行排障效率。

## 推荐方案

- 在 `OperationsDetailPanel` 内维护本地筛选状态：`all`、`tool`、`failure`。
- 运行中心 Agent 运行时间线增加筛选按钮：
  - 全部
  - 工具
  - 失败
- 时间线按筛选状态展示事件，并显示匹配数量。
- 在 Agent 运行时间线卡片中展示最近模型请求线索：请求 ID 和模型名称。

## 边界

- 本阶段不新增后端 DTO、不新增 DDL、不改变 Agent Runtime API。
- “最近模型请求线索”是项目级最近请求，不声明为某个 Agent 事件的强外键关系。
- 筛选只影响运行中心展示，不影响右侧概要指标。
- 不展示完整 prompt、完整模型响应、API Key 或数据库密码。

## 验收标准

- 运行中心默认展示全部 Agent 事件。
- 点击“工具”后只展示 `TOOL_TRACE`，不展示 `RUN_FAILED`。
- 点击“失败”后只展示失败事件，不展示工具 trace。
- 运行中心展示 `模型请求线索 request-1 · matrixcode-local-product`。
- 桌面端全量、桌面构建、服务端全量和真实集成继续通过。
